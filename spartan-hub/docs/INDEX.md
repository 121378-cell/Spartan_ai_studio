# Índice de Documentación Spartan Hub

Esta carpeta centraliza toda la documentación del proyecto. A continuación se detalla la estructura y contenido de cada sección.

## Estructura de Directorios

### 🤖 [AI](ai/)
Documentación relacionada con los servicios de Inteligencia Artificial, implementación de modelos y manejo de errores de IA.
- `AI_ERROR_HANDLING_IMPLEMENTATION.md`: Manejo de errores en el servicio de IA.
- `AI_MESSAGE_QUEUE_IMPLEMENTATION.md`: Sistema de colas para mensajes de IA.
- `AI_REALTIME_FEEDBACK_IMPLEMENTATION.md`: Feedback en tiempo real.

### 🔌 [API](api/)
Especificaciones y guías de las APIs del sistema.
- `FITNESS_NUTRITION_APIS.md`: APIs de nutrición y fitness.
- `RATE_LIMITING_SUMMARY.md`: Implementación de limitación de tasa.

### 🏗️ [Architecture](architecture/)
Decisiones arquitectónicas, patrones de diseño y mejores prácticas.
- `FINAL_SOLUTION.md`: Solución arquitectónica final.
- `STRUCTURED_LOGGING.md`: Guía de logging estructurado.
- `DATA_PERSISTENCE_README.md`: Estrategias de persistencia de datos.

### 🚀 [Deployment](deployment/)
Guías de despliegue, configuración de entorno y Docker.
- `DEPLOYMENT_README.md`: Guía general de despliegue.
- `STARTUP.md`: Instrucciones de inicio.
- `POSTGRESQL_MIGRATION.md`: Migración a PostgreSQL.

### 📚 [Guides](guides/)
Guías detalladas para desarrolladores y usuarios sobre características específicas.
- `INTEGRATION_GUIDE.md`: Guía de integración.
- `UI_ENHANCEMENT_GUIDE.md`: Mejoras de interfaz de usuario.

### ⚙️ [Implementation](implementation/)
Detalles técnicos de implementación de características específicas.
- `INPUT_SANITIZATION_IMPLEMENTATION.md`: Sanitización de entradas.
- `CACHE_SYSTEM_IMPLEMENTATION.md`: Sistema de caché.
- `ERROR_REPORTING_IMPLEMENTATION.md`: Reporte de errores.

### 🔒 [Security](security/)
Documentación de seguridad, auditorías y correcciones.
- `SECURITY_ENHANCEMENT_SUMMARY.md`: Resumen de mejoras de seguridad.
- `INPUT_SANITIZATION_SECURITY.md`: Seguridad en sanitización.

### 🛠️ [Setup](setup/)
Instrucciones de configuración inicial y entorno local.
- `SETUP_INSTRUCTIONS.md`: Instrucciones de configuración.
- `LOCAL_SETUP.md`: Configuración local.
- `API_KEY_SETUP.md`: Configuración de claves API.

## Documentación General y Auditoría (Raíz de docs/)

Archivos adicionales movidos desde la raíz del proyecto que cubren auditorías, registros de sesiones y documentación varia:
- **Auditorías**: `AUDITORIA_PROFUNDA.md`, `RESUMEN_AUDITORIA.md`, `INDICE_AUDITORIA.md`
- **Progreso y Sesiones**: `SESION_PROGRESS_*.md`, `SESION_FIXES_*.md`
- **Resúmenes**: `RESUMEN_FINAL.md`, `RESUMEN_ERRORES_RAPIDO.md`
- **Agentes**: `AGENTS.md` (Guías para agentes de código)

---
*Nota: Esta estructura ha sido centralizada para facilitar el acceso y mantenimiento de la documentación.*
