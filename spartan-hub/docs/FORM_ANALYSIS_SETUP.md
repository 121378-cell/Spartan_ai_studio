# Spartan Hub 2.0 - Video Form Analysis Setup

This document describes how to set up and use the AI-powered video form analysis system.

## 🚀 Overview

The system uses MediaPipe Pose detection to analyze exercise form in real-time. It currently supports:
- **Squats**: Depth detection, knee tracking (valgus), and back angle.
- **Deadlifts**: Back straightness, hip hinge, and lockout detection.
- **Push-ups**: Depth, body alignment, and elbow flaring.

## 🛠️ Frontend Setup

The frontend components are located in `src/components/FormAnalysis/`.

### Key Components:
- `VideoCapture.tsx`: Manages WebRTC camera stream and adaptive resolution.
- `PoseOverlay.tsx`: Renders the skeleton and joints on top of the video.
- `FormAnalysisModal.tsx`: The main orchestration modal.
- `FormScoreCard.tsx`: Displays results, metrics, and recommendations.
- `FormTrends.tsx`: Visualizes progress over time.

### Custom Hook:
- `useFormAnalysis.ts`: Handles the analysis loop, calibration, and backend persistence.

## 💾 Backend Setup

The backend integration handles data persistence and ML risk forecasting.

### Key Files:
- `backend/src/services/formAnalysisService.ts`: Business logic for processing analyses.
- `backend/src/database/migrations/012-create-form-analysis-tables.ts`: Database schema.
- `backend/src/services/mlForecastingService.ts`: Updates injury risk based on form quality.

## 📱 Mobile Optimization

The system automatically detects device type and adjusts:
- **Desktop**: 720p HD resolution.
- **Tablet**: 480p resolution.
- **Mobile**: 480x640 Portrait resolution for better framing.

## 🤖 AI Coaching

Quantitative feedback is generated by geometric analyzers in `src/services/analyzers/`. 
Qualitative feedback is optionally enhanced by `GeminiVisionService` (mocked in MVP).
